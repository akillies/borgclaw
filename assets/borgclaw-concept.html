The Assimilation Flow
$ ./assimilate.sh
🦾 BORGCLAW ASSIMILATOR v1.0

1. Detect Hardware

Auto-detects OS (macOS / Linux / Windows), CPU architecture, GPU type and VRAM, total RAM. No input needed. Takes ~2 seconds.
Detected: macOS / arm64 / Apple M4 Pro / 24GB

2. Classify → Hardware Profile

Maps hardware to capability tier:
HEAVY 32GB+ AND discrete GPU → 14B+ reasoning
STANDARD 16-31GB → 7-9B general purpose
LIGHT 8-15GB → 3B or embeddings-only
MICRO <8GB → embeddings-only node
Profile: STANDARD → Model: qwen2.5:7b

3. Install Ollama + Pull Models

One-line Ollama install (skips if present). Pulls the optimal model for this hardware profile. Always pulls nomic-embed-text (274MB, useful on every node). Optional: pre-staged model files on the USB skip the download entirely.
$ ollama pull qwen2.5:7b ... done (4.4GB)
$ ollama pull nomic-embed-text ... done (274MB)

4. Configure LAN Access

Sets OLLAMA_HOST=0.0.0.0 so the node is reachable from any machine on the home network. Restarts the Ollama service.

5. Register with the Queen

POSTs a registration payload to the Queen's endpoint with hostname, IP, port, profile, models, capabilities, and hardware specs. Queen updates its routing table. Heartbeat begins (every 60s).
✅ NODE ASSIMILATED
Host: alexs-macmini (192.168.1.10:11434)
Profile: STANDARD | Models: qwen2.5:7b, nomic-embed-text
Queen: 192.168.1.10:8000 | Status: ONLINE

6. Ongoing: Heartbeat + Auto-Recovery

Node pings Queen every 60 seconds with status and current load. Queen removes silent nodes after 5 minutes. Nodes auto-reconnect on reboot via a system service. Walk away — it just works.
Total time: ~5 minutes per machine

The Queen — Central Orchestrator

The Queen lives on the always-on primary node. She maintains a live registry of all nodes, their capabilities, and their current load. She routes tasks intelligently — local first, external only when needed.

Routing Rules

task = triage / routing → Fastest node (lowest latency)
task = code / reasoning → HEAVY node (best model) fallback: STANDARD → external API
task = writing / voice-match → External API (frontier model) only case we leave the house
task = embeddings → ANY node (all have nomic-embed) prefer least-loaded
task = monitoring / scheduled → ALWAYS-ON node only
all nodes busy → Queue (don't go external) alert if queue > threshold
