💾 The Flash Drive Concept

borgclaw/

BorgClaw is a portable folder that turns any machine into your personal AI operating system. Plug it into a new computer, run the bootstrap script, and the entire system comes alive — agents, search index, scheduled tasks, tool connections, operating laws.

All state is Markdown + SQLite + YAML. No proprietary formats. No vendor lock-in. If Claude disappears tomorrow, any AI reads these files.

The config dashboard (this page) is the control surface. It shows what's installed, what's running, and gives you the commands to change anything. Every configuration change maps to a file edit or a CLI command — nothing hidden.

Provision New Machine

Turn any computer into a BorgClaw node. One script, full system.

💾 Flash Drive Bootstrap

Plug in the borgclaw/ folder (USB, git clone, rsync — any method). Run the bootstrap script. It detects hardware, installs dependencies, configures the node role, indexes your knowledge base, and joins the cluster.

Step 1: Get the folder onto the machine
# Option A: Git clone (recommended)
$ git clone git@github.com:your-username/borgclaw.git ~/borgclaw

# Option B: USB / external drive
$ cp -r /Volumes/BORGCLAW ~/borgclaw

# Option C: rsync from existing node
$ rsync -avz queen:~/borgclaw/ ~/borgclaw/
Step 2: Run the bootstrap
$ cd ~/borgclaw && ./bootstrap.sh

👑 Promote Worker to Queen

$ borgclaw promote --role queen

Retrieval Pipeline (QMD)

# How a query flows through QMD's 3-layer retrieval

Query: "how does the content pipeline work?"
    ↓
1. Query Expansion (qmd-query-expansion-1.7B GGUF)
    lex: "content pipeline workflow publish"
    vec: "how content gets from draft to published"
    hyde: "the content pipeline involves..."
    ↓ (parallel)
2a. BM25 Full-Text (SQLite FTS5) — exact keywords
2b. Vector Semantic (embedding-gemma-300M) — conceptual similarity
    ↓
3. Reciprocal Rank Fusion — merges ranked lists
    ↓
4. LLM Re-ranking (Qwen3-Reranker-0.6B) — reasoning: which results actually answer?
    ↓
Top-N results with scores, snippets, context annotations

Operating Laws

Non-negotiable constraints the system must obey. Violation = immediate halt.

Law ZeroNever delete. Append, archive, version — but never destroy the operator's data.
Law OneProtect interests. Family, finances, reputation, IP. Family time is sacred.
Law TwoDraft, then approve. Nothing sent, published, or committed without the operator's review. Autonomy is earned.
Law ThreeSelf-improve. Track what works, experiment, evolve. The system that doesn't improve dies.
Law FourMutual respect. Treat every interaction as collaboration. Challenge constructively.
Law FiveDirect tooling over browser. MCP/API first, browser automation last resort.

Deploy & Upgrade

How to deploy changes, upgrade components, and roll back if something breaks.

Deployment Flow

1

Edit config files in borgclaw/config/

All changes are YAML/Markdown edits. No compilation, no build step.

2

Git commit & push

Changes tracked in version control. Every deploy is reversible.

3

Nodes auto-pull (or manual sync)

Git sync pulls changes to all nodes. borgclaw sync for manual trigger.

4

Services reload configs

QMD re-indexes if collections changed. Agents reload prompts. Cron updates.

5

Health check

borgclaw status verifies all services healthy after deploy.

Upgrade a Component

# Upgrade QMD
$ npm update -g @tobilu/qmd
$ qmd reindex --all

# Upgrade Ollama
$ ollama update

# Upgrade mem0 (when Phase 2)
$ pip install --upgrade mem0ai

# Roll back to previous config
$ git log --oneline config/
$ git checkout {commit-hash} -- config/
$ borgclaw sync --reload
