End-to-End: Task → Execution → Delivery

How a single to-do becomes a published article, social posts, and a podcast script — with parallel agents and human approval gates.

Operator creates card in Fizzy: "Publish content platform article on AI job exposure" Fizzy webhook fires → n8n workflow triggers n8n Node 1: Jarvis classifies task → type: "workflow" → template: "content-publish-pipeline" → decomposes into 6 subtasks n8n Node 2: Cerebro researches topic → Queries MCP Gateway Registry: "web search tools" → Discovers web_search MCP dynamically → Loads context from LanceDB: Interest Ontology + signals → Produces: research_brief.md → Publishes event to NATS: events.task.step_complete n8n Node 3 (PARALLEL): ├── Comms-Drafter writes article (voice rules from LanceDB) ├── Comms-Drafter writes social posts (X, LinkedIn, Threads) └── Comms-Drafter writes podcast script n8n Node 4: APPROVAL GATE → ntfy push to operator's phone: "✅ Review: content draft ready" [Approve] [Reject] [View Draft] → Operator taps "Approve" → Queen API receives approval → Workflow resumes n8n Node 5 (PARALLEL): ├── n8n Substack node → publishes article ├── n8n LinkedIn node → posts social ├── n8n Twitter node → posts to X └── Ops-Handler generates header image via Canva MCP n8n Node 6: Cleanup → Updates Fizzy card → COMPLETE (via API) → Logs to NATS: events.task.completed → Updates knowledge base STATE.md → ntfy: "Published: AI Job Exposure Analysis" Total time: ~15 min (mostly waiting for operator's approval tap)

Meanwhile, in parallel...

Sentinel (always running, separate workflow): → Scans inbox every 30 min → Detects expert network email (key contact) → Publishes to NATS: events.signal.expert_network → ntfy CRITICAL alert: "Key contact replied! Expert network revenue." → Creates Fizzy card: "Reply to key contact within 24 hours" Cerebro (morning briefing, separate workflow): → Scheduled 8:30 AM → Scans web for signals across 11 interest domains → Scores each signal: Domain × Cross-domain × Actionability → Drafts morning intelligence brief → Creates Gmail draft (via n8n Gmail node) → ntfy: "Morning brief ready" All workflows run independently, in parallel, across nodes.

Self-Improvement System

Inspired by Karpathy's autoresearch (36K stars): modify → test → measure → keep/discard → loop forever.

Every component of BorgClaw has a measurable metric, a baseline, an experiment budget, and a binary keep/discard rule. The system evolves itself.

The Autoresearch Pattern

LOOP FOREVER: 1. Read current state (experiment logs, metrics, baselines) 2. Propose ONE change to ONE surface 3. Run bounded experiment (fixed time/task budget) 4. Measure result (single metric per surface) 5. If improved → KEEP (update baseline) If not → DISCARD (revert, log result) 6. Append to experiments.tsv (never delete — Law Zero) 7. NEVER STOP. If stuck, think harder.
